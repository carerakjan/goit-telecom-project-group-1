Розгортання проєкту Прогнозування Відтоку Клієнтів для Телекомунікаційної компанії

Цей документ описує кроки, необхідні для розгортання проєкту прогнозування відтоку клієнтів телекомунікаційної компанії.
Проєкт використовує Docker для контейнеризації. Проєкт розгортається в локальному або в хмарному середовищі (Azure).


1. Системні вимоги
Операційна система: Windows 10 або новіша (рекомендується Windows Server для робочого середовища)
Docker Desktop для Windows: Версія 4.0 або новіша
Docker Compose: Вбудований у Docker Desktop
Обліковий запис Azure: З правами на створення віртуальних машин та керування ресурсами


Секція 1: Локальне розгортання на Windows

1.1 Передумови

1. Переконайтеся, що на вашому вашій системі встановлені:
Docker Desktop для Windows
Docker Compose (вбудований у Docker Desktop)

1.2 Встановлення Docker Desktop

1. Завантажте Docker Desktop з офіційного сайту Docker Desktop for Windows.
2. Встановіть Docker Desktop, дотримуючись інструкцій інсталятора.
3. Переконайтеся, що опція Enable WSL 2 Windows Features встановлена, якщо ви плануєте використовувати WSL 2.

1.3. Налаштування Docker Desktop

1. Після встановлення, запустіть Docker Desktop.
2. Переконайтеся, що Docker працює, виконавши команду:
docker --version
docker-compose --version

вв1.4 Запуск локального проєкту

1. Перехід до директорії проєкту:
Відкрийте PowerShell або Command Prompt перейдіть до директорії, де знаходиться ваш проєкт:
cd шлях\до\проєкту
2. Запуск Docker Compose:
Відкрийте PowerShell або Command Prompt у директорії проєкту де є файл docker-compose.yml та виконайте команду:
docker-compose up -d
3. Перевірка роботи додатка:
Перейдіть за адресою http://localhost:8501 (або відповідний порт, вказаний у вашому docker-compose.yml),
щоб перевірити, чи додаток працює коректно.
4. Очищення ресурсів,
Щоб зупинити та видалити контейнери, створені Docker Compose, виконайте наступну команду в PowerShell або Command Prompt:
docker-compose down

1.5 Керування ресурсами Docker

1. Зупинка контейнерів,
Щоб тимчасово зупинити роботу всіх контейнерів, запущених через Docker Compose, використовуйте наступну команду
в PowerShell або Command Prompt у директорії проєкту:
docker-compose stop
Ця команда призведе до зупинки всіх активних контейнерів, що дозволить за потреби швидко відновити їх роботу
без потреби перезапускати й реініціалізувати всі сервіси.
2. Видалення контейнерів
Коли потрібно не тільки зупинити, але й повністю видалити всі контейнери, що були створені Docker Compose,
скористайтеся командою:
docker-compose down
Ця команда зупинить всі контейнери, які були запущені через Docker Compose, і видалить їх разом
з асоційованими мережами та анонімними томами.
Якщо потрібно також видалити всі зображення, використовуйте:
docker-compose down --rmi all

1.6 Видалення специфічних контейнерів

1. Зупинка контейнера:
docker stop [CONTAINER_ID або NAMES]
2. Видалення контейнера:
docker rm [CONTAINER_ID або NAMES]

1.7 Перевірка стану контейнерів

1. Для перевірки, які контейнери зараз запущені, використовуйте:
docker ps
2. Для перегляду всіх контейнерів, включаючи зупинені, використовуйте:
docker ps -a


Секція 2: Розгортання проєкту в Azure

2.1. Підготовка віртуальної машини (VM) на Azure

1. Створення віртуальної машини:
Увійдіть до свого облікового запису на Azure Portal.
Виберіть "Create a resource" та виберіть "Compute" -> "Virtual Machines".
Налаштуйте віртуальну машину, вибравши тип інстансу (наприклад, B2s), регіон, ім'я та
операційну систему (наприклад, Windows Server 2019).
На етапі налаштування диска виберіть розмір та тип диска.
Встановіть та налаштуйте публічний IP і відкрийте необхідні порти (наприклад, порт 8501)
Клацніть "Review + create" і після перевірки натисніть "Create".

2. Встановлення Docker на віртуальну машину:
Підключіться до вашої віртуальної машини через Remote Desktop.
Відкрийте PowerShell або Command Prompt як адміністратор і встановіть Docker  виконавши команду:
Install-Module -Name DockerMsftProvider -Repository PSGallery -Force
Install-Package -Name docker -ProviderName DockerMsftProvider
Перезавантажте віртуальну машину і перевірте, що Docker встановлено виконавши команду:
docker --version

2.2 Розгортання проєкту

1. Завантаження та Розгортання проєкту:
Використовуючи Remote Desktop, завантажте проєкт на віртуальну машину або клонуйте його з Git.
Для запуску проєкту відкрийте PowerShell або Command Prompt в директорії проєкту і виконайте команду:
docker-compose up -d

2. Доступ до проєкту:
Відкрийте веб-браузер і введіть IP-адресу віртуальної машини, вказану на порталі Azure.
Переконайтеся, що відповідні порти (наприклад, 8501 для Streamlit) відкриті у налаштуваннях мережевої безпеки Azure.

3. Очищення ресурсів
Завершіть роботу Docker Compose командою docker-compose down.
Видаліть віртуальну машину на Azure Portal, якщо вона більше не потрібна, щоб уникнути зайвих витрат.
